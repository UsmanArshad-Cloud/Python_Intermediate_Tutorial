Inheritance:

  • The constructor of the superclass is not automatically called in the subclass. You need to call it explicitly using super() if needed.
  • The destructor of the subclass is called upon deletion, but the destructor of the superclass is not automatically executed. It's advisable to avoid using explicit destructors and rely on Python's garbage collection.

Multi-Level and Multiple Inheritance is allowed in Python

Multi-threading:
Multi-threading in Python is a concurrent programming technique that allows you to execute multiple threads (smaller units of a process) concurrently within a single program.

Thread1=threading.thread(target=func)

Thread1.start()=>to start a thread

Thread.join is used for waiting 4 threads to complete so that other lines of code can be executed.

DrawBacks of Multiple threads:

While multithreading is useful for certain tasks, it's essential to be aware of potential pitfalls, such as thread synchronization issues, race conditions, and the Global Interpreter Lock (GIL) in CPython, which can limit true parallelism for CPU-bound tasks. For CPU-bound tasks, you might consider using the multiprocessing module for parallelism or other asynchronous programming techniques such as asyncio for I/O-bound tasks.

Thread Synchronization:

Locking Mechanism:

Locking the resource so that other threads can't access it until first thread complete its working

Semaphore:

Is the mechanism in which the we limit the number of threads that can access the shared resource

Events:

In Python, an "Event" is a synchronization primitive provided by the threading module. It is used for communication between threads to signal that a particular event has occurred.

For example Racers are waiting for the signal to start the race.

Daemon Threads:

These threads are the special threads that always runs in the background and do not wild with the program i.e.if there is one or more non-daemon threads running then the program will wait for these threads to complete gracefully but on the other hand if the some daemon threads are also running but they are not completed gracefully the program will forcefully shut them down when non-daemon threads completed.

Sockets:

Sockets are just like the end points of the communication like the client and the server.It is used for the network communication b/w different or same device via internet or other local network

Server Socket:

To create a server, you need a socket object that listens for incoming connections. The server socket is used to establish a connection with clients.

Client Socket:

A client socket is used to initiate a connection with a server. Clients connect to a specific IP address and port number to establish communication.

Binding and Listening (Server):

For a server socket, you typically bind it to a specific IP address and port number using the bind() method. Then, you call the listen() method to start listening for incoming connections.

Connecting (Client):

For a client socket, you use the connect() method to establish a connection with the server.

Accepting Connections (Server):

Once a server is listening, it can accept incoming connections using the accept() method. This method returns a new socket representing the connection and a tuple with the address of the client.

Sending and Receiving Data:

After a connection is established, both the client and server can use the send() and recv() methods to send and receive data over the connection.

Closing the Connection:

After the data exchange is complete, both the client and server should close the connection using the close() method.
